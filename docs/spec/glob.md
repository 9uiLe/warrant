# 仕様: fnmatch 互換のパスマッチング（glob）

`internal/glob` は、`test_globs` によるテストファイル探索と、`derived_globs` による派生データ判定の両方で使うパスマッチングを提供する。Python の `fnmatch` および再帰 glob と互換のセマンティクスを持つ。

## マッチング規則

`FnMatch(name, pattern)` は Python `fnmatch` 互換で、次のように変換した正規表現で全体一致（`^`…`$`）を判定する。

- `*` は `/` を含め任意の文字列にマッチする（DOTALL）。
- `?` は任意の 1 文字にマッチする。
- `[...]` は文字クラス。先頭の `!` は否定（`^`）に変換する。
- 上記以外の文字はリテラルとしてエスケープする。
- 大文字・小文字は区別する（`*.TXT` は `Hello.txt` にマッチしない）。

## 再帰 glob

`Glob(root, pattern)` は `root` を起点に、forward-slash の相対パスを返す。

- `**` はフルセグメントのときのみ再帰し、0 個以上のディレクトリにマッチする。
- ワイルドカードを含むセグメントは、先頭がドットの名前（`.git`、`.warrant` など）にはマッチしない。
- 返却パスは常に forward-slash 区切りに正規化する。
